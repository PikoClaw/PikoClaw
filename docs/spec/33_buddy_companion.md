## Overview

Every PikoClaw user gets a unique virtual companion — a tiny ASCII creature that lives in the TUI beside the input bar. It animates while the agent thinks, reacts to events (tool errors, successful edits, rate limits), and has its own name and personality generated by the AI on first hatch.

The companion is **deterministic**: given the same machine identifier, you always get the same creature. It's never stored visually — only the name and personality are saved. This means you can't cheat your rarity tier by editing a config file.

PikoClaw's version extends the original concept with:
- **Mood system** — companion reacts to what's happening in the session
- **Level progression** — companion gains XP from tool calls, unlocks accessories
- **`/buddy` commands** — pet, rename, show stats, mute
- **Ratatui-native rendering** — proper widget, no React dependency

## PRNG: Mulberry32 (port to Rust)

```rust
pub struct Mulberry32 {
    state: u32,
}

impl Mulberry32 {
    pub fn new(seed: u32) -> Self {
        Self { state: seed }
    }

    pub fn next_f64(&mut self) -> f64 {
        self.state = self.state.wrapping_add(0x6d2b79f5);
        let mut t = (self.state ^ (self.state >> 15)).wrapping_mul(1 | self.state);
        t = t.wrapping_add((t ^ (t >> 7)).wrapping_mul(61 | t)) ^ t;
        ((t ^ (t >> 14)) as f64) / 4294967296.0
    }

    pub fn next_usize(&mut self, max: usize) -> usize {
        (self.next_f64() * max as f64) as usize
    }
}
```

### Seeding

```rust
const SALT: &str = "pikoclaw-companion-v1";

pub fn seed_from_machine_id() -> u32 {
    // macOS:  IOPlatformUUID from `ioreg -rd1 -c IOPlatformExpertDevice`
    // Linux:  /etc/machine-id
    // Fallback: hash of username + hostname
    let id = get_machine_id().unwrap_or_else(|| whoami::username() + &whoami::hostname());
    fnv1a_32((id + SALT).as_bytes())
}

fn fnv1a_32(data: &[u8]) -> u32 {
    let mut hash: u32 = 0x811c9dc5;
    for byte in data {
        hash ^= *byte as u32;
        hash = hash.wrapping_mul(0x01000193);
    }
    hash
}
```

Using machine ID (not user account) means the companion is tied to the machine, not an account — better for a local CLI tool with no auth.

## Rolling Algorithm

```rust
pub fn roll(seed: u32) -> CompanionBones {
    let mut rng = Mulberry32::new(seed);

    // 1. Rarity
    let rarity = roll_rarity(&mut rng);

    // 2. Species
    let species = SPECIES[rng.next_usize(SPECIES.len())];

    // 3. Eyes
    let eye = EYES[rng.next_usize(EYES.len())];

    // 4. Hat (none if Common)
    let hat = if rarity == Rarity::Common {
        Hat::None
    } else {
        HATS[rng.next_usize(HATS.len())]
    };

    // 5. Shiny (1%)
    let shiny = rng.next_f64() < 0.01;

    // 6. Stats
    let stats = roll_stats(&mut rng, rarity);

    // 7. Inspiration seed (for AI soul generation)
    let inspiration_seed = (rng.next_f64() * u64::MAX as f64) as u64;

    CompanionBones { rarity, species, eye, hat, shiny, stats, inspiration_seed }
}

fn roll_rarity(rng: &mut Mulberry32) -> Rarity {
    let roll = rng.next_f64() * 100.0;
    match roll as u32 {
        0..=59  => Rarity::Common,
        60..=84 => Rarity::Uncommon,
        85..=94 => Rarity::Rare,
        95..=98 => Rarity::Epic,
        _       => Rarity::Legendary,
    }
}

fn roll_stats(rng: &mut Mulberry32, rarity: Rarity) -> Stats {
    let floor = rarity.stat_floor();  // Common=5, Uncommon=15, Rare=25, Epic=35, Legendary=50
    let peak = STAT_NAMES[rng.next_usize(5)];
    let mut dump = STAT_NAMES[rng.next_usize(5)];
    while dump == peak { dump = STAT_NAMES[rng.next_usize(5)]; }

    STAT_NAMES.iter().map(|&name| {
        let value = if name == peak {
            ((floor + 50) as f64 + rng.next_f64() * 30.0).min(100.0) as u8
        } else if name == dump {
            ((floor as f64 - 10.0 + rng.next_f64() * 15.0).max(1.0)) as u8
        } else {
            (floor as f64 + rng.next_f64() * 40.0) as u8
        };
        (name, value)
    }).collect()
}
```

## Soul Generation (First Hatch)

On first launch with no `[companion]` in config, call the API to generate the soul:

```rust
pub async fn generate_soul(
    bones: &CompanionBones,
    api_client: &AnthropicClient,
) -> CompanionSoul {
    let prompt = format!(
        "You are naming a new virtual companion. \
         It is a {species} with {rarity} rarity. \
         Its peak stat is {peak_stat}. Inspiration seed: {seed}.\n\
         Reply with exactly two lines:\n\
         Name: <one creative name>\n\
         Personality: <one sentence, max 10 words>",
        species = bones.species,
        rarity = bones.rarity,
        peak_stat = bones.peak_stat_name(),
        seed = bones.inspiration_seed,
    );

    // Use haiku for speed and cost
    let response = api_client.quick_message("claude-haiku-4-5", &prompt).await
        .unwrap_or_default();

    parse_soul_response(&response).unwrap_or_else(|| CompanionSoul {
        name: "Chip".to_string(),
        personality: "Reliable and quietly enthusiastic".to_string(),
        hatched_at: Utc::now(),
    })
}
```

## ASCII Sprite System

Each species has 3 animation frames, 5 lines tall, 12 chars wide. `{E}` is replaced with the companion's eye character.

### Example: Duck

```
Frame 0:          Frame 1:          Frame 2 (blink):
            __
  _  _     /  \       _  _              _  _
 ({E}{E})  ({E}{E})   ({-}{-})   ← blink on frame -1
  (  )     (  )       (  )
 /    \   /    \     /    \
```

### Hat Placement

Hat goes on line 0 of the sprite. If line 0 is blank, it gets replaced:

```
Hat::Crown:      "   \\^^^/   "
Hat::Tophat:     "   [___]   "
Hat::Propeller:  "    -+-    "
Hat::Halo:       "   (   )   "
Hat::Wizard:     "    /^\\    "
Hat::Beanie:     "   (___)   "
Hat::Tinyduck:   "    ,>     "
```

### Shiny Rendering

Shiny companions render with a sparkle color (bright yellow/gold in theme) instead of normal text color.

### Idle Animation Sequence

## Mood System (PikoClaw Extension)

The companion reacts to session events. Mood affects the speech bubble text shown beside the sprite.

```rust
pub enum Mood {
    Idle,         // default — no speech bubble
    Thinking,     // agent is processing
    ToolRunning,  // bash/file tool in progress
    Success,      // tool completed successfully
    Error,        // tool returned error
    RateLimit,    // rate limit hit
    Excited,      // long successful turn (many tools)
    Sleepy,       // session idle > 5 min
}
```

### Speech Bubbles Per Mood

## Level & XP System (PikoClaw Extension)

Companion gains XP from session activity. Levels unlock cosmetic accessories.

```rust
pub struct CompanionProgress {
    pub level: u32,      // starts at 1
    pub xp: u32,         // current XP in this level
    pub xp_to_next: u32, // XP needed for next level
}
```

### XP Sources

| Event | XP |
|-------|----|
| Successful bash command | +2 |
| File edit completed | +3 |
| Successful agent turn | +5 |
| Multi-tool turn (3+ tools) | +10 |
| Error recovered (tool failed then succeeded) | +8 |

### Level Unlocks

| Level | Unlock |
|-------|--------|
| 1 | Base companion |
| 3 | Companion shows name in TUI |
| 5 | Unlocks one random hat from pool |
| 10 | Speech bubbles become longer/more expressive |
| 20 | Shiny effect activates on companion |
| 50 | Secret "veteran" sprite frame |

Level-up shows a one-time message: `✨ {name} reached level {n}!`

## TUI Widget

The companion renders as a ratatui `Widget` in a small panel beside the input bar (or in the status bar area).

```
┌─ Input ────────────────────────────┐  ╔══╗
│ >                                  │  ║{E}{E}║  ← Ferris (Lv.3)
└────────────────────────────────────┘  ║(  )║
                                        ╚══╝
```

Layout options (configurable):
- `right` — small panel to the right of the input bar (default)
- `status` — tiny inline face in the status bar (`[🦆 Ferris]`)
- `off` — hidden but still tracking XP


## Edge Cases

- Machine ID unavailable: fall back to `$USER@$HOSTNAME` hash — companion may differ across machines, acceptable
- Soul generation API call fails on first run: use deterministic fallback names based on species (`Lucky the Duck`, `Sage the Owl`, etc.)
- `/buddy reroll` explicitly blocked with message: "Your companion is permanent. Every `{name}` is unique to this machine."
- Muted companion still accumulates XP — level-up shown on next unmute
- Companion in status bar mode (`layout = "status"`): show `[🦆 L7]` — species emoji + level
- Terminal doesn't support Unicode/emoji: fall back to ASCII face `[^.^]`
